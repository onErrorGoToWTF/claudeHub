# data/ schema reference

One-page contract for every JSON shape that `js/app.js` or the fetchers consume. Update this doc whenever you change a shape. Schema versions bump with backwards-incompatible changes.

---

## `data/version.json`

Written by each milestone commit. `js/app.js` reads this to populate the footer.

```json
{
  "version":       "0.6.115",
  "milestone":     "M8.12.45",
  "milestoneTitle": "Phase 8.12.45 — …",
  "deployedAt":    "2026-04-19T23:15Z",
  "commitSha":     ""
}
```

Fields:
- `version` — semver string (major.minor.patch). Bumped on every milestone commit.
- `milestone` — short code (`M{phase}.{sub}.{step}` or `M9.N` under the v0.7 refactor).
- `milestoneTitle` — one-line change summary, free text.
- `deployedAt` — ISO timestamp (minute resolution OK).
- `commitSha` — optional, filled by CI if wired; otherwise empty string.

## `data/latest.json`

Auto-generated by `scripts/build_latest_json.js` every 2h (GitHub Actions cron). Never hand-edit.

```json
{
  "generated_at": "2026-04-19T21:48:39.411Z",
  "version": 1,
  "sections": {
    "updates":         [FeedItem],
    "news":            [FeedItem],
    "status":          [StatusItem],
    "youtube":         [FeedItem],
    "tutorials":       [TutorialItem],
    "claude_learning": [FeedItem]
  }
}
```

**FeedItem** (shared across `updates`/`news`/`youtube`/`claude_learning`):
```
{
  "title":     string,
  "url":       string,
  "source":    string,      // human-friendly producer label
  "published": ISO8601,
  "summary":   string        // may be empty
}
```

News items may carry `_kind: "news_video"` when a YouTube clip is routed into news (per-rules in `build_latest_json.js`).

**StatusItem** — same as FeedItem but `source` is the status page.

**TutorialItem**: FeedItem + `tutorial_kind: "video" | "official"`.

**Gotchas**:
- `merge()` in the orchestrator preserves prior values when a fetch returns empty; the site never blanks on transient fetch failure.

## `data/learn/lessons.json`

Hand-authored lessons with inline MCQ quizzes. `body` markdown lives at `data/learn/lessons/<slug>.md`.

```json
{
  "schema_version": 1,
  "note": "…",
  "lessons": [LessonEntry]
}
```

**LessonEntry** (post-M9.1 migration):
```
{
  "slug":         string,             // URL-safe unique id
  "title":        string,
  "track":        "claude-code"|"skills"|"mcp"|"finder"|"image-video"|"nocode"|"local",
  "order":        number,
  "minutes":      number,             // read time estimate
  "summary":      string,
  "status":       "authored"|"draft", // M9.1 migration auto-sets "authored" for existing entries
  "learnPrompt":  string|null,        // M9: filled when pinned-from-Tools created the stub
  "notes":        string|null,        // M9: optional user notes at pin time
  "docsUrl":      string|null,        // M9: external docs link for draft stubs
  "sourceToolId": string|null,        // M9: references a tool id when created via pin-to-learn
  "quiz":         [QuizItem]          // may be empty
}
```

**QuizItem**:
```
{
  "q":        string,
  "choices":  [string, string, string],
  "answer":   number,    // 0-indexed into choices
  "why":      string     // explanation shown after reveal
}
```

## `data/learn/tools.json`

Curated tool catalog. Hand-edited.

```json
{
  "updated_at": "2026-04-19",
  "tools": [ToolEntry]
}
```

**ToolEntry**:
```
{
  "id":              string,           // URL-safe unique id
  "name":            string,
  "vendor":          string,
  "modality":        "llm"|"coding"|"image"|"video"|"voice"|"3d"|"automation"|"deploy"|"data"|"motion-ui",
  "tagline":         string,
  "whenToUse":       string,
  "currentVersion":  string,
  "pricing":         string,
  "priceTier":       "free"|"lte20"|"lte100"|"lte200"|"gt200"|"api",
  "claudeNative":    boolean,
  "setupComplexity": 1|2|3|4|5,
  "priorityScore":   number,            // 1..5, Finder weight
  "provides":        [string],          // capability IDs
  "requires":        [string],          // capability IDs
  "officialUrl":     string,
  "docsUrl":         string,
  "topics":          [string],
  "newsKeywords":    [string],          // lowercase, used by news filter
  "verifiedAt":      "YYYY-MM-DD",
  "snippetTags":     [string],          // matches snippets.snippetTags
  "setupSnippet":    string             // multi-line bash/text
}
```

## `data/learn/snippets.json`

Reusable code/config snippets. Hand-edited.

```json
{
  "schema_version": 1,
  "note": "…",
  "snippets": [SnippetEntry]
}
```

**SnippetEntry**:
```
{
  "id":           string,
  "title":        string,
  "summary":      string,
  "tags":         [string],          // freeform
  "snippetTags":  [string],          // fixed vocabulary — see note in file
  "language":     "markdown"|"bash"|"yaml"|"json"|"typescript"|"python"|"jsx"|…,
  "body":         string              // multi-line, uses literal \n in JSON
}
```

Fixed `snippetTags` vocabulary (as of M8.12): `claude-md`, `slash-command`, `hook`, `output-style`, `mcp-config`, `skill-yaml`, `prompt-image`, `prompt-video`, `prompt-system`, `n8n-workflow`, `prompt-nocode`, `sdk-anthropic`, `code-python`.

## localStorage keys (browser-side, no server schema)

All under the `clhub.v1.*` namespace. Bumped only on breaking shape changes.

- `clhub.v1.projects`       — `{ [projectId]: SavedProject }`
- `clhub.v1.lessonProgress` — `{ [slug]: { state: "new"|"in_progress"|"completed", quizPassedAt?: ISO8601 } }`
- `clhub.v1.finderDraft`    — current in-flight Finder state (single slot): `{ description: string, path?: "easy"|"best", updatedAt: ISO8601 }`. Pre-M9.5b entries stored just the description string; `getFinderDraft` migrates them transparently on read. A Resume banner appears at the top of `#projects/new` whenever `description.trim()` is non-empty; **Start fresh** clears the slot + the separate `finderCaps` store.
- `clhub.v1.finderCaps`     — last capability-box selections
- `clhub.v1.mastery`        — `{ [key]: ISO8601 }` where `key` is `lesson:<slug>`, `course:<slug>`, `tool:<toolId>`, or `draft:<draftId>` *(M9.4b — unified "I've got this" signal; swipe-left on a Learn card sets it + unpins; Tools catalog renders ●ᴹ badge for mastered tools in M9.7)*
- `clhub.v1.learnDrafts`    — `[{ id, learnPrompt, notes, docsUrl, sourceToolId, createdAt }]`. *(M9.6b — user-created draft lesson stubs pinned from Tools cards; rendered in Learn's Everything else zone with a Draft pill. A future LLM hydrator could promote stubs into authored lessons; the schema is forward-compatible with Q13.)*
- `clhub.v1.ui`             — misc UI prefs (selected filters, active sort)

**SavedProject** (when M9 settles):
```
{
  "id":          string,
  "title":       string,
  "path":        "easy" | "best",
  "stack":       [{ toolId: string }],
  "pinnedTools": [string],
  "pinnedSnippets": [string],
  "notes":       string,
  "createdAt":   ISO8601,
  "updatedAt":   ISO8601
}
```
