aiUniversity — Supabase schema

Reference visualization of the schema drafted in supabase-schema.sql (same folder). Tables split into shared content, user-scoped, and scaffold-for-later. All user-scoped tables enforce ownership via Row-Level Security: every query is auto-filtered to user_id = auth.uid().

Shared content (read-only to users, seeded from repo) User-scoped (RLS-owned) Scaffolded for friend-view, unused in v1 PK primary key FK foreign key RLS row-level security

Cascade summary

Knowing what happens when you delete something matters more than you think.

delete auth.users row→ cascades to every user_*** table (user_profile, user_progress, user_mastery, user_projects, user_project_events, user_library_state, user_search_misses, friendships). Account wipe.
delete user_projects row→ cascades to its user_project_events. Project history vanishes with the project.
delete track / topic / lesson→ RESTRICT. Blocks if any dependent rows exist. Prevents an accidental content delete from nuking user progress.
delete library_items row→ cascades to user_library_state for that item (pin/own state lost). Content authors should prefer unpublishing (body = null) over deleting.

Source of truth: supabase-schema.sql in this folder. If anything here diverges, the SQL wins.
