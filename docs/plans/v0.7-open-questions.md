# v0.7 design-system refactor — open questions + defaults

Every unresolved question from the plan review, with my proposed default. **Read top-to-bottom. Reply only to items you want to change.** Silence = default stands.

Format: `Q. question — **Default: X.** Why: Y.`

---

## Color system

1. Dashboard `--base` — stay greige or pick final now? — **Default: `#9a938a` (current), revisit at M9.3.** Delaying costs nothing; Dashboard components aren't identity-tinted until that milestone.
2. Keep Learn/Projects/Tools hexes unchanged? — **Default: yes, unchanged.** `color-mix` derivation works on any hex; the recent retunes stand.
3. `--color-365-*` / `--accent-365-*` cleanup — **Default: rename to `--accent-violet-*`.** Still referenced by unrelated gradients; renaming preserves those without deleting.
4. `MODEL_COL` chart palette — **Default: stays independent of `--accent-*`.** Model colors are brand identity (Anthropic orange ≠ accent orange); coupling them would mis-tint charts.
5. `color-mix(in oklch)` browser floor — **Default: accept, no sRGB fallback.** iOS 16.4+ is the target; 2023 shipped.
6. `--accent-warm` / `--accent-cool` recipes now? — **Default: define them — `color-mix(in oklch, var(--base) 82%, #ffc400)` warm, `color-mix(in oklch, var(--base) 82%, #5ab8ff)` cool.** Locked recipes cascade; tuning later is a one-line change.
7. Dashboard CTA edge — **Default: neutral `--border-strong`, not colored.** Dashboard is the overview; identity color belongs to tab pages.

## Learn

8. Up Next cap of 3 — **Default: hard cap at 3 with `+N more pinned` affordance that sends extras to Everything-else.** Preserves "today" feel; no item loss.
9. "Everything else" default sort — **Default: Recent.** Matches running-feed feel; A–Z one tap away.
10. Quiz-only lesson (no body) shows? — **Default: yes, shows; card type pill reads "Quiz" not "Lesson".** Useful surface area.
11. Filter chip "Pinned" scope — **Default: global (shows pinned items regardless of zone).** Filter is about source/nature, not location.
12. Swipe-left consequences — **Default: moves to Done, 5s toast with Undo.** Consistent with Done bucket; Undo guards the gesture.
13. Draft lesson schema — **Default: `{ status: "draft"|"authored", learnPrompt: string, notes?: string, docsUrl: string, sourceToolId: string|null }`.** Forward-compatible with a future LLM hydrator.
14. Mastery localStorage — **Default: `clhub.v1.mastery = { [toolId]: ISO8601 }`.** Matches existing `clhub.v1.*` namespace; timestamp enables "mastered 3 days ago" later.
15. Academy courses in unified list — **Default: yes, included.** Flattening is the whole point; hiding scraped items defeats it.

## Projects

16. Detail view edit model — **Default: live-edit, write-through to localStorage.** Matches existing notes-pad autosave; fewer clicks.
17. Route `#projects/new` — **Default: extend chip router to recognize `#<chip>/<view>`.** Preserves the existing system; direct-linkable; back-button works.
18. Finder draft storage — **Default: single slot `clhub.v1.finderDraft`, most recent wins.** Multi-draft adds UX burden you don't want.
19. `+N more` stack chip tap — **Default: expand in place.** Less disruptive than opening detail.
20. Saved card tap target — **Default: whole card opens detail.** Single tap target is iPhone ergonomic best practice; quick actions live behind long-press / `⋯`.
21. Filter "Easy"/"Best" field — **Default: existing `project.path: "easy"|"best"`.** Already in schema.
22. Bootstrap ghost tap — **Default: toast reads "Bootstrapper arrives in a later milestone — your stack + pins seed it." No telemetry.** No backend.

## Tools

23. "Mastered skills" bucket placement — **Default: top-of-page horizontal strip above the catalog, labeled "Your stack".** Visual weight without fragmenting into a sub-tab.
24. `●ᴹ` mastery badge placement — **Default: title-line suffix on the tool card.** Stays in glance-zone.
25. Pin-to-learn modal fields — **Default: two fields — "What do you want to learn?" (required) + "Notes" (optional).** Friction kills pin rate.

## Cross-cutting

26. Pill radius — **Default: absolute 6px for all non-true-pill shapes. Genuine circles (avatars, spinners, dots) use a fixed px, not `9999`.** Kills ambiguity; the "true pill" category is empty.
27. `--radius-pill: 9999px` token — **Default: delete.** Rule 26 empties its use cases; deletion prevents accidental re-use.
28. Ambient radial overlay intensity — **Default: stay at current alphas; revisit after panels lighten in M9.3.** Evaluate on live site.
29. Glass-tile "fake hole" fill — **Default: `--bg-0` literally.** One fewer token; glass-edge chrome sells the depth.
30. Impressed label — **Default: single `.label-debossed` utility class applied where needed.** One place to tune.

## Workflow

31. Milestone numbering — **Default: open `M9` for the v0.7 refactor; use `M9.1`–`M9.11` for the eleven steps.** Closes the `M8.12.X` cursor cleanly at `v0.6.115-pre-refactor`.
32. Commit gating — **Default: one commit per visible delta (~11 commits, occasionally two per milestone).** Matches phone-review cadence.
33. `lessons.json` migration — **Default: one-time build-script migration during M9.1 auto-adds `status: "authored"` to existing entries.** No hand-editing.
34. Prompt-before-remove rule scope — **Default: user-visible strings only. CSS class names, tokens, internal identifiers don't trigger.** Implementation tidy doesn't stall.

---

## Resolutions to the 5 flagged risks

- **Pill ambiguity** → resolved by Q26 + Q27 (6px everywhere; delete the 9999 token).
- **Dashboard-as-canonical with grey `--base`** → resolved by scoping `--accent-*` to `[data-section="X"]`. A Dashboard component cloned onto Learn takes Learn's `--base` via descendant selector; Dashboard itself stays neutral.
- **Cascade vs overrides collision** → **Default: overrides live in `css/overrides.css` (new file), each with a one-line comment naming the reason.** Single place to audit; prevents per-file drift.
- **`--color-365-*` orphan pre-refactor break** → resolved by Q3 rename happening BEFORE the token refactor (M9.1 stage 1).
- **Finder `#projects/new` silent resume** → **Default: auto-resume if draft exists, but show a visible `Resume draft` banner with a `Start fresh` link beside it.** No silent resurrection.
