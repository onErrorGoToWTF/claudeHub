## Milestones (M9.1 → M9.11)

Each milestone ends with: commit (conventional prefix) → bump `data/version.json` → push → phone review.

### M9.1 — Token system foundation
One commit. Zero visible delta.

- Introduce `--base` CSS var pattern per page via `[data-section="X"] { --base: ...; }`.
- Derive `--accent-solid / border / glow / surface / ink / hover / soft / warm / cool` through `color-mix(in oklch, ...)`.
- Refactor ~40–60 hardcoded `--color-{learn,projects,tools,youtube}-*` references to the new token system. Legacy tokens stay aliased so nothing breaks while references migrate.
- Add `css/overrides.css` as the single home for intentional per-component deviations (file created even if empty).
- One-time `lessons.json` migration: auto-add `status: "authored"` to every existing lesson.
- Commit: `refactor(M9.1): introduce --base + --accent-* token system [v0.7.0-dev.1]`.
- **No phone pause** — zero visible change; continue to M9.2.

### M9.2 — Pill-radius global purge
One commit. Low-risk visible polish.

- Replace every non-circle `border-radius: 9999px` / `var(--radius-pill)` with `var(--radius-sm)` (6px).
- Delete the `--radius-pill` token definition.
- Genuine circles (avatars, spinners, dots) use explicit `border-radius: 50%` or a fixed px, never the removed token.
- Commit: `style(M9.2): purge pill radius — 6px for all pill-shaped elements [v0.7.0-dev.2]`.
- **Phone pause #1** after push.

## Post-sweep activation polish (M9.12 – M9.15a) — shipped

Incremental polish on top of the v0.7.0 stamp. Each bullet is a live commit; `data/version.json` milestone + version fields track the exact cut. All rules captured in CLAUDE.md design-language (animation sequencing, transforms are exclusive, no flicker on repaint, prefer static / spatial solutions) plus the matching memory entry `feedback_animation_sequencing.md`.

- **M9.12 [v0.7.1-dev.1]** — `feat`: Dashboard panels get a scroll-center activation (IntersectionObserver toggles `.is-activated` in a vertical center band). Tile baseline bg lifts off the page tone toward white so text contrast is stronger at rest.
- **M9.13 [v0.7.1-dev.2]** — `feat`: Replace the per-tile bg animation with a fixed `html::after` vignette overlay (z-index 5, pointer-events none, warm-greige gradient dim at top/bottom, transparent through the center). Spatial effect, no animation — tuneable in isolation. Activation observer extended to `.learn-zone`. Widen band to ~50% (rootMargin -25%).
- **M9.13a [v0.7.1-dev.3]** — `style`: Band tightens to ~35% (rootMargin -32.5%). Vignette peak alpha 0.13 → 0.24 with 3-stop falloff.
- **M9.14 [v0.7.1-dev.4]** — `feat`: Unify the active-highlight recipe across nav chips + filter chips + activated panels (thin `--accent-border` + tight `--accent-glow`). Collapse all per-section `--base` overrides to a single `--base-shared` token at `:root` (temporarily `#ff7a1a` tangerine); inline `/* was #XXX */` comments preserve each original literal for later restoration.
- **M9.14a [v0.7.1-dev.5]** — `style`: Drop the inset 1px inner ring from the five unified highlight recipes so the line reads roughly half-weight. Dashboard CTAs + drag-drop drop-candidate keep their inset rings (different components, intentional punch).
- **M9.15 [v0.7.1-dev.6]** — `feat`: Activation distributes to the interactive bits inside an activated panel. Panel surfaces no longer paint accent (too much color on the big surface — explicit override of the unified recipe). Dashboard CTAs go flat/neutral at rest and only light accent border + inset ring + outer glow inside an activated panel. Dashboard icons (`.dash-panel-icon--*`) go greige `rgba(30,24,16,0.22)` stroke + `filter: none` at rest and reclaim their stroke + drop-shadow glow only in-band. Tiles inside activated panels raise up with a coordinated shadow. `--base-shared` flips from tangerine to `#84cc16` warm vibrant green. Band tightens to ~25% (rootMargin -37.5%).
- **M9.15a [v0.7.1-dev.7]** — `style`: Band tightens to ~20% (rootMargin -40%). Same model; narrower observer window.

## Failure recovery

1. User sends screenshot + problem description.
2. Agent: `git diff HEAD~1 <affected-file>` → identify the change → ONE surgical fix.
3. If the fix doesn't land after second attempt, STOP. Reassess root cause with user input. Do not ship a third patch.
